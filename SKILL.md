# Japan Video Editor

Automated pipeline for producing Japanese-language vertical shorts (TikTok / Instagram Reels / YouTube Shorts) from raw iPhone footage. Outputs a 60–120s vertical video with Japanese captions, English subtitles, BGM, SFX, and optional B-roll.

## When to Use

Activate this skill when:
- User drops a video file or says "edit this video"
- User says "new video project", "run the pipeline", "start from scratch"
- User is working in a project that has a `raw/` folder with `.mp4` files

## Project Folder Structure

Every project follows this layout:

```
<project-name>/
├── raw/          ← source clips from iPhone (.mp4) — DROP HERE
├── segments/     ← trimmed segments cut by ffmpeg (auto-generated)
├── sfx/          ← SFX files (.mp3/.wav) — pre-loaded, shared
├── bgm/          ← background music (.mp3/.wav) — pre-loaded, shared
├── broll/        ← B-roll clips (.mp4, auto-generated by Genspark)
├── output/       ← final rendered videos (final_v1.mp4, final_v2.mp4 …)
├── public/       ← Remotion static assets (symlinks to above dirs)
│   ├── segments/ → ../segments/
│   ├── sfx/      → ../sfx/
│   ├── bgm/      → ../bgm/
│   └── broll/    → ../broll/
├── src/
│   ├── index.ts              ← Remotion entry point
│   ├── VideoEdit.tsx         ← main composition (auto-generated/updated)
│   └── components/
│       └── DualSubtitle.tsx  ← bilingual JP/EN subtitle component
├── scripts/
│   └── silence_cut.py        ← Step 1.5 silence auto-cut preprocessor
├── edit_plan.json       ← structured edit plan (auto-generated)
├── transcript.json      ← Whisper output (auto-generated)
├── package.json
└── tsconfig.json
```

## The 10-Step Pipeline

Run steps in order. Each step's output is the next step's input.

### Step 1 — Convert source clips to MP4

```bash
# For each clip in raw/ that isn't already h264/mp4:
ffmpeg -i raw/INPUT.MOV -c:v libx264 -crf 18 -preset fast -c:a aac raw/INPUT.mp4
```

All clips must be in `raw/` as `.mp4` before Step 1.5.

### Step 1.5 — Silence auto-cut

```bash
# Run on every raw clip; output lands in segments/
for f in raw/*.mp4; do
  python scripts/silence_cut.py "$f" "segments/$(basename "${f%.mp4}")_cut.mp4"
done
```

Default settings (threshold −30 dB, min silence 0.4 s, pad 0.08 s) work for ~80% of clips. See `references/silence-cut.md` for per-environment tuning.

Output: `segments/<basename>_cut.mp4` — one silence-trimmed file per raw clip.

### Step 2 — Transcribe with Whisper

```python
import whisper, json
model = whisper.load_model("small")
# Transcribe the silence-cut segments (not the raw clips)
all_segments = []
for clip in sorted(Path("segments").glob("*_cut.mp4")):
    result = model.transcribe(str(clip), language="ja", word_timestamps=True)
    all_segments.extend(result["segments"])
Path("transcript.json").write_text(json.dumps(all_segments, ensure_ascii=False, indent=2))
```

Output: `transcript.json` — list of segment objects with `start`, `end`, `text`, `words`.

### Step 3 — Transcript quality check

Run the **whisper-quality-checker** sub-task. It fixes:
- Filler words (えーと、あのー、えっと)
- Duplicate phrases / stutters
- Real estate kanji errors (e.g. 礼金 misread as 例金)

```
delegate_task(
    goal="Check and fix transcript.json for a Japanese real estate video",
    context="Fix filler words, duplicates, and real estate kanji misreads. Input: transcript.json. Output: transcript.json (in-place).",
    toolsets=["file", "terminal"]
)
```

### Step 4 — Generate edit_plan.json

Run the **video-content-planner** sub-task. It reads `transcript.json` and produces `edit_plan.json`.

Rules for the planner:
- Target 60–120s total duration (never over 90s for Reels)
- Speed: 1.3× (keep_segments only — BGM and SFX play at 1×)
- Resolution: 1080×1920 (9:16 vertical)
- Captions: use `text_jp` (≤15 Japanese chars) + `text_en` (≤8 words, concise translation)
- Tag first segment `style: "hook"`, last segment `style: "cta"`, on-screen labels `style: "label"`
- Highlight 30–40% of captions (key nouns, numbers, action phrases)
- Insert B-roll at topic transitions (use `broll_slots`)
- Map SFX to highlighted captions (see `references/sfx-library.md`)
- Include one BGM track from `bgm/`

See `references/edit-plan-schema.md` for the full JSON schema.

### Step 5 — Generate VideoEdit.tsx

Use `edit_plan.json` to write `src/VideoEdit.tsx`. 

See `references/remotion-template.md` for the full template.

Key customisations per project:
- Import `DualSubtitle` from `./components/DualSubtitle` (replaces old `Subtitle`)
- `CAPTION_SFX`: caption id → sfx filename (from edit_plan)
- `BROLL_SLOTS`: array from edit_plan broll_slots
- `CAPTION_SFX_VOLUME`: per-caption volume overrides (default 0.6)
- `text_en` in `edit_plan.json` drives EN subtitles automatically — no separate `ENGLISH` map needed

### Step 6 — Wire public/ symlinks

```bash
cd <project-dir>
ln -sf ../segments public/segments  # or ../segments if already at project root
ln -sf ../bgm public/bgm
ln -sf ../broll public/broll
ln -sf ../sfx public/sfx
```

Then copy / cut segments from raw clips:

```bash
# For each keep_segment in edit_plan.json:
ffmpeg -ss SOURCE_START -to SOURCE_END -i raw/SOURCE_FILE \
  -c:v libx264 -crf 18 -preset fast -c:a aac \
  segments/seg_NN.mp4
```

Segment filenames must be `seg_00.mp4`, `seg_01.mp4`, … (zero-padded to 2 digits).

### Step 7 — Preview in Remotion Studio

```bash
npm start
```

Opens browser at `http://localhost:3000`. Scrub through composition to verify visuals, captions, BGM timing.

### Step 8 — Quality verification (up to 3 iterations)

Run **video-quality-verifier** sub-task. Checks:
1. Black frames (>0.5s) — fail if found
2. Audio pops / clipping — fail if peak > -1 dBFS
3. Caption sync (captions vs. speech within 0.3s)
4. Resolution exactly 1080×1920
5. Audio levels (BGM −18 to −12 dBFS, voice −6 to −3 dBFS)
6. Caption positioning (bottom 18%, never overlapping face)
7. Total duration ≤ 90s

See `references/quality-checks.md` for full criteria and fix procedures.

If verifier fails, fix the specific issue and re-run (max 3 iterations).

### Step 9 — Export final render

```bash
npx remotion render src/index.ts JapanVideoEdit output/final_v1.mp4 \
  --codec=h264 --crf=18
```

### Step 10 — (Optional) CapCut fine-tuning

Import `output/final_v1.mp4` into CapCut Desktop for:
- Manual caption position adjustments
- Additional sticker/transition effects
- Final audio normalisation

## Key Production Rules

| Parameter | Value |
|-----------|-------|
| Resolution | 1080 × 1920 (9:16) |
| FPS | 30 |
| Playback speed | 1.3× (segments only) |
| Caption max length | 15 Japanese chars |
| Highlight ratio | ~35% of captions |
| Target duration | 60–90s |
| BGM volume | 0.08–0.12 |
| SFX default volume | 0.6 |
| Caption font | Meiryo / Yu Gothic, 44px bold |
| Caption position | bottom 18% |
| Caption fade | 3 frames in / 3 frames out |

## Setting Up a New Project

1. Copy the `educ-re-1` folder as a template:
   ```bash
   cp -r ~/educ-re-1 ~/<new-project-name>
   rm -rf <new-project-name>/raw/* <new-project-name>/segments/* \
          <new-project-name>/output/* <new-project-name>/transcript.json \
          <new-project-name>/edit_plan.json <new-project-name>/src/VideoEdit.tsx
   ```
2. Drop source clips into `raw/`
3. Start the pipeline at Step 1

## Troubleshooting

**Remotion shows black/missing media** → public/ symlinks are broken or segments/ is empty. Re-run Step 6.

**Whisper misreads real estate terms** → Step 3 (quality check) handles this. Common: 礼金→例金, 保証→補償, 仲介→中介.

**Audio pops at segment boundaries** → add `-af "afade=t=in:d=0.05,afade=t=out:d=0.05"` to ffmpeg in Step 6.

**Caption overflows 15 chars** → split into two captions in edit_plan.json and adjust start/end times.

**Silence-cut clips speech** → increase `--pad` (try 0.12–0.15) in `scripts/silence_cut.py`.

**Video still feels slow after silence cut** → lower `--min-silence` to 0.3 or raise `--threshold` to −25 for noisy environments.

**EN subtitle renders too large** → check the `style` field in `edit_plan.json`; `label` segments must omit `text_en`.
